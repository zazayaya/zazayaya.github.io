Linux-wireguard安装文档

2021年7月12日
| Linux 网络
| 阅读
文章目录

架构

默认架构

典型应用：实现两台服务器之间互相访问：https://www.wireguard.com/quickstart/

两台服务器均创建 wireguard 类型的虚拟设备，通过将物理网卡的数据流转到虚拟设备进行数据认证和过滤

############ 创建 wireguard 类型的虚拟设备 ############
# 1、创建私钥、公钥
wg genkey | tee privatekey | wg pubkey > publickey
# 2、创建 wireguard 类型的虚拟设备
ip link add dev wg0 type wireguard
# 3、添加 ip，不要和本机 ip 冲突
ip address add dev wg0 10.0.0.1/24
# 4、加载私钥
wg set wg0 private-key ./privatekey
# 5、启动虚拟设备
ip link set wg0 up
# ping 对方的隧道的设备 ip
# ping: sendmsg: Required key not available  # 这里说明这个设备的IP是特有协议的，需要key才能访问


############ 分别建立对等隧道 ############
# 对端服务器执行 wg 命令 可以查看 key 和 端口
# endpoint 的 ip 是物理网卡的 ip，或者是外面 IP、映射 IP
wg set wg0 peer drCvrSFi+ZSPu9p4PBxLh702RVyGBa7kcl/umqxLpz8= allowed-ips 10.0.0.2/32 endpoint 192.168.5.191:45317
# wg set wg0 peer drCvrSFi+ZSPu9p4PBxLh702RVyGBa7kcl/umqxLpz8= remove

转发架构

典型应用：做为 vpn 服务端来应用，可以数据转发

实现：默认架构 + iptables 的 nat 转发

转发架构

服务端

简单理解：数据转发还是通过防火墙的nat实现，数据通过wireguard虚拟设备进入(主要用于认证和过滤)

前置条件

# 开启转发
sysctl -p | grep -q 'net.ipv4.ip_forward = 1' || (echo net.ipv4.ip_forward=1 >> /etc/sysctl.conf && sysctl -p)

# INPUT需要开启虚拟网卡访问的权限
iptables -A INPUT -i wg0 -j ACCEPT

安装

ubuntu 20.04已添加 WireGuard 支持，直接安装即可，安装包：wireguard、wireguard-tools

ubuntu 18.04安装包：wireguard、wireguard-dkms、wireguard-tools，20.04默认打进内核所以不需要安装：wireguard-dkms

# ubuntu 20.04,依赖包自动被安装：wireguard-tools
apt update && apt -y install wireguard 

# ubuntu 18.04
# apt install software-properties-common
# add-apt-repository ppa:wireguard/wireguard
# apt install wireguard  # 包含：wireguard、wireguard-dkms、wireguard-tools

生成私钥、公钥

wireguard 类型的虚拟网卡需要加载密钥

WireGuard 包含两个工具，名为wg和wg-quick，它允许你配置和管理 WireGuard 接口

# 设置掩码权限
umask 077
# 生成私钥
[ -f /etc/wireguard/privatekey ] || wg genkey > /etc/wireguard/privatekey
# 生成公钥
[ -f /etc/wireguard/publickey ] || wg pubkey < /etc/wireguard/privatekey > /etc/wireguard/publickey
# 也可以一条命令生成
# wg genkey | tee privatekey | wg pubkey > publickey

配置虚拟设备

网卡名：名称为转发的目标网卡名

privatekey=$(cat /etc/wireguard/privatekey)
interface=$(ip -o -4 route show to default | awk '{print $5}')
[ -f /etc/wireguard/wg0.conf ] || cat > /etc/wireguard/wg0.conf << EOF
[Interface]
# 这个 ip 是 vpn 的网关 ip，可以自定一个网段来使用，注意不要和本地 ip 冲突了
Address = 192.168.26.1/24
SaveConfig = true
ListenPort = 51826
# PrivateKey = SERVER_PRIVATE_KEY值（cat /etc/wireguard/privatekey）
PrivateKey = $privatekey
# 这里需要保证网关的网卡名不能错误
# PostUp = iptables -A FORWARD -i %i -j ACCEPT; iptables -t nat -A POSTROUTING -o 网卡名 -j MASQUERADE
# PostDown = iptables -D FORWARD -i %i -j ACCEPT; iptables -t nat -D POSTROUTING -o 网卡名 -j MASQUERADE
PostUp = iptables -A FORWARD -i %i -j ACCEPT; iptables -t nat -A POSTROUTING -o $interface -j MASQUERADE
PostDown = iptables -D FORWARD -i %i -j ACCEPT; iptables -t nat -D POSTROUTING -o $interface -j MASQUERADE
EOF

# 说明：
# [Interface]：服务器端配置开始标志
# Address：为VPN网段网关地址，注意只能是24位掩码，这里做为 VPN 网段
# SaveConfig：为true时，有客户端连接时，保存状态信息，退出时自动保存到wg0.conf文件
# PrivateKey：为服务器的私钥
# ListenPort：为服务器监听的端口号，默认：51820

# 查看PostUp、PostDown网卡名： ip -o -4 route show to default | awk '{print $5}'
# 网络不通日志调试
# A POSTROUTING -o eth0 -j LOG --log-prefix "IPTABLES POSTROUTING: "

启动虚拟设备

chown -v root:root /etc/wireguard/wg0.conf
chmod -v 600 /etc/wireguard/wg0.conf
# 启动 WireGuard interfaces
wg-quick up wg0
# 查看 WireGuard interfaces 信息
wg
# 查看虚拟卡信息
ip addr show wg0
# 开机自启动
systemctl enable wg-quick@wg0.service

# wg-quick up wg0 执行过程日志如下：
[#] ip link add wg0 type wireguard
[#] wg setconf wg0 /dev/fd/63
[#] ip -4 address add 192.168.26.1/24 dev wg0
[#] ip link set mtu 1420 up dev wg0
[#] iptables -A FORWARD -i wg0 -j ACCEPT; iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE

# 关闭虚拟网卡
# wg-quick down wg0
# 启动虚拟网卡
# wg-quick up wg0

增加对等隧道(添加用户)

注意：ip不能冲突，会导致相关ip用户断网

# vpn 外部网络可以访问的 ip
username="zaza"			   # 每个用户都需要修改
user_ip="192.168.26.2/32"  # 每个用户都需要修改为唯一，否则相关ip的公钥会被替换掉
Peer_AllowedIPs="192.168.26.1/32,2.2.2.2/32"    # 这个是客户端路由的地址
pub_ip="1.1.1.1"		   # 通常为服务器或者公司外网ip
# 生成客户端 ip
# 实现自动添加 ip
# vpn_gw=$(ip addr show wg0 | grep 'scope global wg0' |awk '{print $2}')
# 访问限制,2.2.2.2/32 为需要转发的 ip
wg_AllowedIPs=${user_ip},2.2.2.2/32  # 这个是vpn客户端可以访问的地址

################################
mkdir /etc/wireguard/users
# 生成私钥
user_pri=$(wg genkey)
# 生成公钥
user_pub=$(echo $user_pri | wg pubkey)
# 服务器公钥
ser_pub=$(cat /etc/wireguard/publickey)


# 服务器添加用户对端隧道配置
wg set wg0 peer $user_pub allowed-ips $wg_AllowedIPs
wg-quick save wg0

# 生成客户端配置
[ -f /etc/wireguard/users/${username}.conf ] || cat > /etc/wireguard/users/${username}.conf << EOF
[Interface]
PrivateKey = $user_pri
Address = ${user_ip}
DNS = 114.114.114.114

[Peer]
PublicKey = ${ser_pub}
AllowedIPs = ${Peer_AllowedIPs}
Endpoint = ${pub_ip}:51826
PersistentKeepalive = 25
EOF

# [Peer]：客户端开始标志，（多个客户端，会有多个配置连接）
# PublicKey：为客户端的公钥，这个需要客户端生成，客户端生成完成后，需要填写
# AllowedIPs：
#	允许客户端连接的地址，注意多用户，必须使用32位掩码
#	客户端用于添加路由，对端用于对数据进行过滤

# 二维码生成，方便手机客户端使用
# qrencode -t ansiutf8 < 1.txt

客户端

WireGuard各系统安装包

创建新隧道

快捷键：ctrl+n

新建隧道(注意点击旁边的倒三角图标) -> 新建空隧道

配置参考

查看用户配置：cat /etc/wireguard/users/${username}.conf

############################
[Interface]
PrivateKey = 客户端的私钥(新建空隧道也会自动生成)
Address = 192.168.26.2/32
DNS = 114.114.114.114

[Peer]
PublicKey = 服务器的公钥（cat /etc/wireguard/publickey）
Endpoint = 服务器的物理ip地址:51826
AllowedIPs = 192.168.26.0/24
PersistentKeepalive = 25
############################

# 说明：
# [Interface]：客户端配置开始部分
# Address：客户端IP地址（也就是本机地址），多用户的时候，这里指定到32位掩码的具体地址
# PrivateKey：客户端私钥
# DNS：服务器IP地址作为DNS
# [Peer]：服务器端的配置信息
# PublicKey：服务器的公钥（服务端生成的server_public_key文件）
# Endpoint：服务器的IP地址及端口号
# AllowedIPs：就是需要走vpn的地址(会添加相关路由)，0.0.0.0/0为允许所有IP地址通讯(全代理)，如果单个IP地址，可以写多个用逗号隔开，默认写客户端的IP地址
# PersistentKeepalive：用来保持连接检查的，默认每过25s会自动检查连通性，如果IP有变化，也是通过这个自动更新endpoint

客户端登录后验证

登录后，接口状态提示"已连接"(这里显示的应该是：Interface的状态)

但是需要注意观察节点的是接收和发送数据是否有流量

如果接收没有流量，代表远端没有连接成功，可以看WireGuard"隧道"旁边的"日志"里面详细信息

类似异常日志：2021-07-12 13:52:04.787: [TUN] [zbx] peer(HuU2…U60s) - Handshake did not complete after 5 seconds, retrying (try 2)

root@k8s-master-01:/etc/wireguard# wg

  listening port: 51826

  endpoint: 10.0.26.28:54622
  allowed ips: 192.168.26.0/24
  latest handshake: 3 seconds ago       # 这个必须要有，才表示建立连接成功
